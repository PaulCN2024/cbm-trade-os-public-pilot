# Supplier Intelligence RFQ Draft Rules

## Purpose

This document defines safe planning rules for supplier RFQ drafts prepared by AI Supplier Intelligence.

RFQ drafts are internal review materials only. They must not be sent automatically, treated as approved supplier communication, or used to create customer quotations without Paul review.

## RFQ Draft Structure

A future RFQ draft should include:

- greeting
- product/request summary
- technical questions
- commercial questions
- packing/loading questions
- lead time/sample questions
- clarification that it is preliminary
- no final commitment

Recommended structure:

```text
Subject: RFQ for [product/category] - preliminary supplier review

Dear [Supplier],

Please help review whether you can produce/supply the following item for preliminary evaluation.

Product/request summary:
- Product:
- Material:
- Drawing/spec:
- Quantity:
- Destination/trade basis if known:

Technical questions:
- ...

Commercial and packing questions:
- ...

This is for preliminary evaluation only. Final specifications and any order decision will be confirmed later.

Best regards,
Paul
```

## Required Supplier Questions By Scenario

### Light steel keel

Ask about:

- material
- thickness
- profile drawing/spec
- unit weight
- MOQ
- packing
- loading quantity
- FOB/local cost
- lead time

### Aluminum/window

Ask about:

- profile system
- drawing review
- glass/accessories
- surface treatment
- installation/sizing responsibility
- production lead time

### Ceiling system

Ask about:

- panel type
- thickness
- suspension system
- packing
- MOQ
- loading

### Custom/facade

Ask about:

- drawing review
- engineering constraints
- wind load/standard
- fabrication capability
- surface treatment
- sample/mockup

## Safety Wording

Every RFQ draft should be marked:

- Draft only
- Not sent
- Human approval required
- Preliminary evaluation only

Recommended safety note:

```text
This RFQ draft is for internal review only. It has not been sent to any supplier. Paul must approve supplier selection, message content, and sending separately.
```

## What RFQ Drafts Must Not Do

RFQ drafts must not:

- promise order
- confirm final specs
- confirm price acceptance
- commit payment
- commit production schedule
- imply supplier has been selected
- imply customer has approved the product
- expose customer confidential data unnecessarily
- send automatically
- create quotation automatically
- create PI/order/payment/production/shipment actions

## Approval Boundary

Paul approval is required before:

- sending any supplier RFQ
- selecting supplier recipients
- sharing customer drawings or sensitive files
- using supplier answers in a customer quote
- relying on supplier lead time, production feasibility, or quality claims

RFQ draft preparation and RFQ sending must remain separate workflow stages.
